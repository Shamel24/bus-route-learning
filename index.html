Machine Learning: Supervised vs Unsupervised Learning

Labeled Data

Supervised Learning

The computer learns from examples that already have the correct answer attached. After studying enough examples, it can guess the answer for brand-new examples it has never seen before.

  • Needs labeled examples; ones that already have the correct answer
  • Good for: sorting data into groups, or predicting a number
  • Real-world uses: catching spam emails, guessing a house's price, recognizing what's in a photo
  • Some tools it uses: Linear Regression, Decision Trees, Neural Networks

Try it yourself: drive the bus and teach it your route

Rename the stores if you want, pick one, and click along the streets to add them to the map. Then drive the bus with the arrow buttons — every store it drives past gets added to your route, in order. Once you've visited a few stores, click "Train Route Model." The computer studies your route and learns a pattern connecting store type + location to the order you visited them. Then it can guess a route for any set of stores — even ones you never drove to yourself!

Unlabeled Data

Unsupervised Learning

The computer looks at data that has no answers attached at all. It finds patterns, grouping things together all by itself.

  • Doesn't need labeled examples
  • Good for: grouping similar things together, simplifying messy data
  • Real-world uses: grouping shoppers with similar habits, finding similarities, sorting news articles by topic
  • Some tools it uses: K-Means, Hierarchical Clustering, PCA

Try it yourself: let the computer find its own route

Just like before, you can rename the store types, pick one, and click along the streets to add stores to the map. But this time you won't drive a route to teach the computer. Instead, click "Predict Route," and the computer will group the stores — all on its own, no example route needed. Then the bus will automatically drive the route, stopping at each store.
